# CrazyBee 官网改版 PRD（V3）

## 1. 项目目标
- 将当前页面升级为可对外展示的品牌官网（Investor / Merchant / User 三端都能看懂）。
- 风格关键词：Apple 式叙事、纯白基底、简约但信息丰满。
- 保留原型预览演示能力，且设备尺寸固定 `393x852`。

## 2. 核心问题
- 当前结构仍偏“原型目录”，品牌与商业信息密度不均衡。
- 顶部导航未完全贴顶并撑满，导航存在“漂浮感”而非“官网系统条”。
- 页面底层仍有红色渐变氛围，不符合“纯白底”的要求。
- 内容模块缺少“市场覆盖”“业务可信度”这类增强信息。

## 3. 用户与场景
- B 端商户：关心平台覆盖、引流能力、数据能力。
- C 端用户：关心优惠强度、下单体验、信任感。
- 投资人/合作方：关心产品定位、规模化路径、完成度。

## 4. 体验原则
- One Screen, One Message：每屏只讲一个核心信息。
- Typographic Hierarchy：大标题、短说明、少量关键点。
- White-first System：页面主背景纯白，仅保留极轻灰层次。
- Evidence-rich：真实餐饮图片 + 覆盖地图 + 原型演示。

## 5. 信息架构（最终页面）
1. Hero（品牌主张 + 主 CTA）
2. Focus（设计与体验方法）
3. Features（4列核心能力）
4. Coverage（美国覆盖地图 + 城市节点）
5. Value（商业价值：Trust/Conversion/Retention）
6. Prototype（可筛选原型演示）
7. Launch（收尾与行动）

## 6. 内容模块说明
### 6.1 Hero
- 主标题：一句价值主张
- 副标题：一句范围+能力说明
- 右侧：真实餐厅场景图

### 6.2 Features
- 采用四列模块（图标+标题+一句解释）
- 与用户给的参考图一致，强调“信息均衡”。

### 6.3 Coverage
- 使用美国地图作为“市场覆盖证明”。
- 标注 Berkeley / Irvine / Bloomington 三个城市。
- 视觉上突出节点，不强依赖复杂地图交互。

### 6.4 Prototype
- 保留筛选：All / Core / Transaction / Account
- iPhone 容器固定 `393x852`，不压缩、不拉伸。

## 7. 视觉规范
- 背景：纯白 `#FFFFFF`
- 主要文字：`#111111`
- 辅助文字：`#666666`
- 品牌红：`#FF3B31`（用于按钮、强调、激活态）
- 去掉大面积红色渐变背景，仅保留纯色与极浅灰分区。

## 8. 交互规范
- 顶部导航：固定贴顶、全宽、激活态红色下划线/胶囊。
- 滚动：章节锚点高亮 + 进度条保留。
- 图片：轻量视差（可选，幅度小）。

## 9. 资产方案（图片）
- 餐饮场景图：Pexels 实拍（商用友好）。
- 覆盖地图：Wikimedia blank US map（CC0）+ 页面标注层。

## 10. 验收标准
- 顶部导航完全贴顶并撑满。
- 页面基底纯白，无红色渐变背景。
- 至少 6 个内容章节，结构清晰且不拥挤。
- 地图+餐饮图+原型三类视觉证据齐全。
- 移动端仍可读，原型区横向滚动可用。
